Forma de uso y trabajo en TensorFlow.js

La forma de trabajo define el marco de desarrollo y uso de este librería, y es por lo tanto, la forma básica y principal de utilización y trabajo en la plataforma. De esta forma, el proceso descrito a continuación puede tomarse como la forma básica de trabajo para la implementación de modelos con TensorFlow.js .


Descripción del proceso

Se distinguen básicamente 3 formas de implementación:

  • Implementación de un modelo entrenado en Python.
  • Implementación de un modelo entrenado en JavaScript.
  • Implementación de un modelo pre entrenado de TensorFlow.js.

  • Resumen en una figura







Modelo entrenado en Python

Básicamente, es el entrenamiento primero en TensorFlow, para después convertir el modelo entrenado a un formato web compatible y de esta forma, portarlo a TensorFlow.js

  1. Diseño y entrenamiento del modelo en lenguaje Python, utilizando la API TensorFlow.
  2. Guardado del modelo en un tipo compatible con convertidor de
    modelos
  3. Conversion del modelo a formato web compatible, utilizando convertidor de modelos.
  4. Implementación del modelo en TensorFlow.js.

    5. Este proceso, que en general resulta ser el más recomendado para aplicaciones finales o modelos en donde se necesita alta capacidad de computo, puede llevarse a cabo con la ayuda del siguiente Notebook de Google Colab :






Modelo entrenado en JavaScript

Es el entrenamiento de principio a fin en TensorFlow.js, con la capacidad de la conversion y guardado a un formato compatible con TensorFlow en Python, para por por ejemplo, entrenar el modelo en una etapa intermedia si es necesaria más capacidad de computo.

  1. Diseño y entrenamiento del modelo en lenguaje JavaScript, utilizando la API TensorFlow.js
  2. Guardado del modelo en formato web compatible.
  3. Implementación del modelo en TensorFlow.js

    4. Este tipo de forma de trabajo es el utilizado en los ejemplos desarrollados para la clasificación del dataset MNIST y el dataset Fashion MNIST.





Modelo pre entrenado de TensorFlow.js

  1. Implementación/carga del modelo en TensorFlow.js
  2. Entrenamiento del modelo en TensorFlow.js.
  3. Guardado del modelo en formato web compatible.
  4. Implementación del modelo en TensorFlow.js

    5. Este tipo de forma de trabajo es el utilizado en el desarrollo del ejemplo Transfer Learning con MobileNet
